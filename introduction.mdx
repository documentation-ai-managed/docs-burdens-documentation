## Overview

Burdens Documentation provides a centralized platform where you organize, manage, and publish all your project documentation. You create structured spaces for APIs, user guides, and onboarding materials using MDX-powered pages with interactive components. Access version control, search, and collaboration features to keep your docs up-to-date and user-friendly.

<Callout kind="info">
Burdens uses the brand color `#3B82F6` for primary elements. Customize themes to match your project's identity.
</Callout>

## Getting Oriented with the Interface

Start by navigating the sidebar for quick access to pages. Use the search bar to find topics instantly. Follow these steps to set up your first space.

<Steps>
  <Step title="Create a New Space" icon="plus">
    Click the `+ New Space` button in the dashboard. Enter a name like `My Project Docs` and select a template.
  </Step>
  <Step title="Add Your First Page" icon="file-text">
    Navigate to your space and select `New Page`. Choose `introduction.mdx` as the file name.
  </Step>
  <Step title="Preview and Publish" icon="eye">
    Edit in the MDX editor, preview changes, then hit `Publish`. Share the live URL with your team.
  </Step>
</Steps>

## Platform-Specific Setup

Configure Burdens for your development environment using these tailored instructions.

<Tabs>
  <Tab title="npm" icon="package">
    Install the CLI tool to manage docs locally.

    <CodeGroup tabs="Install,Configure">
```bash
npm install -g @burdens/cli
```
```javascript
// burdens.config.js
module.exports = {
  brandColor: '#3B82F6',
  spaces: ['./docs'],
};
```
    </CodeGroup>
  </Tab>
  <Tab title="Docker" icon="docker">

````dockerfile

RUN npm install -g @burdens/cli
WORKDIR /docs
CMD ["burdens", "dev"]
````

## Brand Guidelines and Customization

Follow these guidelines to maintain consistency. Customize your docs with the brand color `#3B82F6` and Lucide icons.

<ExpandableGroup>
  <Expandable title="Primary Theme Configuration" default-open="true">
    Update your config file to apply the brand color across all pages.

    ```javascript
    // config/theme.js
    export default {
      colors: {
        primary: '#3B82F6',
        secondary: '#1D4ED8',
      },
      icons: {
        default: 'book-open',
      },
    };
    ```

    Test changes in preview mode before publishing.
  </Expandable>
  <Expandable title="Icon Usage">
    Select from 200+ Lucide icons like `zap`, `shield`, or `rocket`. Avoid custom SVGs for consistency.

    ```jsx
    <Card icon="zap" title="Fast Performance">
      Optimized for quick loads.
    </Card>
    ```
  </Expandable>
</ExpandableGroup>
