## Account Management

Manage your Burdens Documentation account easily with these common solutions.

<Columns cols={2}>
  <Card title="Create Account" icon="user-plus" href="/docs/account/create">
    Sign up in under 2 minutes.
  </Card>
  <Card title="Reset Password" icon="key" href="/docs/account/reset-password">
    Recover access quickly.

<ExpandableGroup>
  <Expandable title="How do I create a new account?">
    Visit the sign-up page at `https://app.burdens.dev/signup`. Enter your email, create a strong password, and verify your email address. You receive instant access to your documentation workspace.
  </Expandable>

  <Expandable title="What if I forget my password?">
    Click "Forgot Password" on the login page. Enter your email to receive a reset link valid for 15 minutes. Follow the steps to set a new password meeting these requirements: at least 12 characters, including uppercase, lowercase, numbers, and symbols.
  </Expandable>
</ExpandableGroup>

## Troubleshooting Access Issues

Resolve common access problems step by step.

<Callout kind="tip">
  Check your internet connection and browser cache first. Clear cache using `Ctrl+Shift+R` (`Cmd+Shift+R` on macOS).
</Callout>

<Steps>
  <Step title="Verify Login Status" icon="check-circle">
    Ensure you use the correct email and password. Enable two-factor authentication (2FA) for added security.
  </Step>
  <Step title="Check Permissions">
    Confirm your role: Admin, Editor, or Viewer. Admins assign roles in workspace settings.
  </Step>
  <Step title="Browser Compatibility">
    Use the latest Chrome, Firefox, or Safari. Disable ad blockers and VPNs temporarily.
  </Step>
  <Step title="Contact Support">
    Submit a ticket via the help menu if issues persist.
  </Step>
</Steps>

## Data Privacy and Security

Burdens Documentation prioritizes your data protection.

| Feature              | Description                                                                 |
|----------------------|-----------------------------------------------------------------------------|
| Encryption           | Data at rest uses AES-256; in transit uses TLS 1.3.                         |
| Access Controls      | Role-based access control (RBAC) with audit logs.                           |
| Compliance           | GDPR, SOC 2 Type II compliant.                                              |
| Data Retention       | You control deletion; automatic purge after 30 days for deleted workspaces. |

<Callout kind="success">
  Enable 2FA and review audit logs weekly to maintain security.
</Callout>

<Expandable title="Where is my data stored?">
  Data resides in secure AWS regions you select during setup (e.g., US East, EU West). No data leaves your chosen region without explicit export.
</Expandable>

## Migrating from Other Tools

Switch to Burdens Documentation seamlessly.

<Tabs>
  <Tab title="From Notion" icon="book-open">
    <Steps>
      <Step title="Export Pages">
        In Notion, select workspace > Export > Markdown & CSV.
      </Step>
      <Step title="Import to Burdens">
        Use the import tool: drag Markdown files into your workspace.
      </Step>
    </Steps>
  </Tab>
  <Tab title="From Confluence" icon="database">
    <CodeGroup tabs="CLI,API">
      ```bash
      # Use Confluence CLI to export
      confluence export-space --space-key MYSPACE --output-dir ./export
      ```
      ```javascript
      // API example
      const response = await fetch('https://your-confluence.atlassian.net/wiki/rest/api/space/MYSPACE/export', {
        headers: { Authorization: 'Bearer YOUR_TOKEN' }
      });
      ```
    </CodeGroup>
    Upload ZIP to Burdens import dashboard.
  </Tab>
  <Tab title="From GitBook" icon="git-branch">
    Export as MDX/ZIP from GitBook settings. Import directly into Burdens workspace via `/import` endpoint.
  </Tab>
</Tabs>

<Expandable title="Will my custom domains migrate?">
  Custom domains require manual DNS updates post-migration. Burdens provides a migration checklist in your dashboard.
</Expandable>

These answers cover most scenarios. For personalized help, contact support at `support@burdens.dev`.
