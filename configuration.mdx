## Theme and Branding

Set your primary color to `#3B82F6` for Burdens branding. Upload logos and adjust fonts via the settings panel.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to **Admin > Configuration > Appearance**.
  </Step>
  <Step title="Update Colors" icon="palette">
    Enter your hex code in the **Primary Color** field.
  </Step>
  <Step title="Upload Assets" icon="upload">
    Add logo and favicon files (PNG, max 2MB).
  </Step>
  <Step title="Preview and Save" icon="eye">
    Review changes and click **Apply**.
  </Step>
</Steps>

<CodeGroup tabs="JSON,YAML">
```json
{
  "theme": {
    "primaryColor": "#3B82F6",
    "logoUrl": "https://your-domain.com/logo.png",
    "fontFamily": "Inter, sans-serif"
  }
}
```
```yaml
theme:
  primaryColor: "#3B82F6"
  logoUrl: "https://your-domain.com/logo.png"
  fontFamily: "Inter, sans-serif"
```
</CodeGroup>

<Callout kind="tip">
  Test theme changes in preview mode before publishing to avoid disrupting users.
</Callout>

## User Roles and Permissions

Assign roles to control access. Admins manage everything, editors update content, and viewers read only.

| Role     | Edit Pages | Manage Users | Delete Content | Export Data |
|----------|------------|--------------|----------------|-------------|
| Admin    | Yes        | Yes          | Yes            | Yes         |
| Editor   | Yes        | No           | No             | Yes         |
| Viewer   | No         | No           | No             | No          |

<Tabs>
  <Tab title="Admin" icon="shield">
    Full access. Invite users and set global permissions.
  </Tab>
  <Tab title="Editor" icon="edit-3">
    Create and update pages. Cannot delete or manage users.
  </Tab>
  <Tab title="Viewer" icon="eye">
    Read-only access. Ideal for external collaborators.
  </Tab>
</Tabs>

## Export and Import Options

Export your site as Markdown or JSON. Import restores from backups.

<CodeGroup tabs="CLI,API">
```bash
# Export to ZIP
burdens export --format=markdown --output=docs-backup.zip

# Import from ZIP
burdens import docs-backup.zip --overwrite=false
```
```javascript
// API endpoint example
const response = await fetch('/api/export', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
  body: JSON.stringify({ format: 'json' })
});
```
</CodeGroup>
